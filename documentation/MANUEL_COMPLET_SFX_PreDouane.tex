\chapter{Introduction}

\section{Présentation de l'Application}

\textbf{SFX PreDouane} est une application web professionnelle de gestion de pré-dédouanement développée par Softronic Innoving. Elle permet aux entreprises de transit et aux commissionnaires en douane de gérer efficacement leurs opérations de dédouanement.

\subsection{Informations Générales}

\begin{itemize}[leftmargin=*]
    \item \textbf{Nom :} SFX PreDouane (SFX Transit)
    \item \textbf{Type :} Application Web
    \item \textbf{Version :} 0.1.0
    \item \textbf{Éditeur :} Softronic Innoving
    \item \textbf{Technologie :} Next.js 16, React 19, Prisma ORM
    \item \textbf{Base de données :} Microsoft SQL Server
\end{itemize}

\subsection{Objectif de l'Application}

L'application SFX PreDouane a pour objectif principal de :

\begin{itemize}[leftmargin=*]
    \item Faciliter la gestion des dossiers de transit douanier
    \item Automatiser le calcul des droits et taxes de douane
    \item Gérer les conversions de devises et les taux de change
    \item Générer automatiquement les notes de détail pour les déclarations
    \item Assurer le suivi des colisages et des marchandises
    \item Optimiser les processus de pré-dédouanement
\end{itemize}

\subsection{Public Cible}

Cette application s'adresse aux profils suivants :

\begin{itemize}[leftmargin=*]
    \item \textbf{Commissionnaires en douane} : Gestion complète des dossiers clients
    \item \textbf{Agents de transit} : Suivi des opérations de transit
    \item \textbf{Responsables d'entités} : Supervision et validation des opérations
    \item \textbf{Administrateurs système} : Configuration et gestion des utilisateurs
\end{itemize}

\section{Fonctionnalités Principales}

\subsection{Gestion des Dossiers}
\begin{itemize}[leftmargin=*]
    \item Création et suivi des dossiers de transit

\section{Contexte Métier}

\subsection{Domaine d'Application}

SFX PreDouane s'inscrit dans le domaine du \textbf{transit douanier} et du \textbf{dédouanement international}. L'application répond aux besoins spécifiques des professionnels du commerce international.

\subsection{Processus Métier}

Le processus typique de pré-dédouanement comprend les étapes suivantes :

\begin{enumerate}[leftmargin=*]
    \item \textbf{Création du dossier} : Ouverture d'un dossier pour un client
    \item \textbf{Saisie du colisage} : Enregistrement des marchandises
    \item \textbf{Association des régimes} : Définition des régimes douaniers applicables
    \item \textbf{Conversion des devises} : Application des taux de change officiels
    \item \textbf{Génération de la note de détail} : Production du document de synthèse
    \item \textbf{Export et transmission} : Envoi aux autorités douanières
\end{enumerate}

\section{Contraintes et Règles Métier}

\subsection{Règles de Gestion des Devises}

\begin{itemize}[leftmargin=*]
    \item Une conversion doit être créée avant de générer une note de détail
    \item Tous les taux de change des devises utilisées doivent être renseignés
    \item La devise locale (ID 0) a toujours un taux de 1.0 (non modifiable)
    \item Les taux de change sont liés à une date de déclaration spécifique
    \item Si une seule devise est présente, elle est automatiquement sélectionnée
\end{itemize}

\subsection{Règles de Gestion des Régimes}

\begin{itemize}[leftmargin=*]
    \item \textbf{100\% DC} : Droits de douane à 100\%
    \item \textbf{100\% TR} : Transit à 100\%
    \item \textbf{EXO} : Exonération totale (taux = 0)
    \item \textbf{TTC} : Toutes taxes comprises (taux = -2)
    \item \textbf{Ratios DC/TR} : Pourcentages personnalisés (ex: 51.49\% DC)
\end{itemize}

\subsection{Règles de Génération des Notes de Détail}

\begin{itemize}[leftmargin=*]
    \item Les notes sont regroupées par régime douanier
    \item Ordre de priorité : 100\% DC, 100\% TR, EXO, Ratios DC, Ratios TR, TTC
    \item Les valeurs sont converties dans la devise de référence sélectionnée
    \item La régénération supprime automatiquement la note existante
    \item Les colonnes sont ordonnées selon les spécifications métier
\end{itemize}

\begin{enumerate}[leftmargin=*]

\section{Gestion des Conversions et Taux de Change}

\subsection{Créer une Conversion}

\textbf{Objectif :} Définir les taux de change pour une date de déclaration.

\textbf{Procédure :}

\begin{enumerate}[leftmargin=*]
    \item Cliquez sur \texttt{Conversions} dans la barre de navigation
    \item Cliquez sur \texttt{+ Nouvelle Conversion}
    \item Sélectionnez la \texttt{Date de conversion}*
    \item Cliquez sur \texttt{Créer}
    \item La conversion est créée, vous pouvez maintenant ajouter les taux
\end{enumerate}

\subsection{Ajouter un Taux de Change}

\textbf{Méthode 1 : Depuis la page de détail de la conversion}

\begin{enumerate}[leftmargin=*]
    \item Ouvrez la conversion concernée
    \item Cliquez sur \texttt{+ Ajouter un taux}
    \item Sélectionnez la \texttt{Devise}*
    \item Saisissez le \texttt{Taux de change}* (avec 3 décimales minimum)
    \item Cliquez sur \texttt{Enregistrer}
\end{enumerate}

\textbf{Méthode 2 : Depuis la liste des conversions (nouveau)}

\begin{enumerate}[leftmargin=*]
    \item Dans la liste des conversions, localisez la conversion
    \item Cliquez sur le bouton \texttt{Ajouter taux} (icône +) dans la colonne Actions
    \item Le dialogue s'ouvre directement
    \item Sélectionnez la devise et saisissez le taux
    \item Cliquez sur \texttt{Enregistrer}
\end{enumerate}

\chapter{Foire Aux Questions (FAQ)}

\section{Questions Générales}

\textbf{Q1 : Puis-je utiliser l'application sur mobile ?}

R : Oui, l'application est responsive et s'adapte aux écrans mobiles. Cependant, pour une expérience optimale et pour les tâches de saisie intensive, l'utilisation sur ordinateur de bureau est recommandée.

\textbf{Q2 : Mes données sont-elles sauvegardées automatiquement ?}

R : Oui, toutes les données sont sauvegardées immédiatement après validation. Il n'y a pas de sauvegarde automatique pendant la saisie, pensez à cliquer sur "Enregistrer".

\textbf{Q3 : Puis-je annuler une action ?}

R : Les suppressions sont irréversibles. Pour les modifications, vous pouvez annuler en cliquant sur "Annuler" avant d'enregistrer. Une fois enregistré, vous devez modifier à nouveau pour corriger.

\section{Questions sur les Dossiers}

\textbf{Q4 : Puis-je modifier un dossier après génération de la note ?}

R : Certains champs peuvent être modifiés, mais les éléments liés au colisage et aux régimes ne peuvent plus être changés après génération de la note de détail. Vous devez supprimer la note pour modifier ces éléments.

\textbf{Q5 : Comment retrouver un dossier rapidement ?}

R : Utilisez la barre de recherche en haut de la liste des dossiers. Vous pouvez rechercher par numéro de dossier, numéro OT, nom de client ou type de dossier.

\textbf{Q6 : Puis-je supprimer un dossier ?}

R : [À compléter : selon les droits utilisateur et les règles métier]

\section{Questions sur les Conversions}

\textbf{Q7 : Pourquoi ne puis-je pas modifier le taux de la devise locale ?}

R : La devise locale (ID 0) a toujours un taux de 1.0 par définition. C'est la devise de référence pour toutes les conversions.

\textbf{Q8 : Puis-je avoir plusieurs conversions pour la même date ?}

R : Non, il ne peut y avoir qu'une seule conversion par date. Si vous devez modifier les taux, modifiez la conversion existante.

\textbf{Q9 : Combien de décimales dois-je saisir pour les taux ?}

R : Saisissez au minimum 3 décimales pour la précision. Le système accepte jusqu'à 6 décimales.

\section{Questions sur la Note de Détail}

\textbf{Q10 : Pourquoi la devise est-elle sélectionnée automatiquement ?}

R : Si votre dossier ne contient qu'une seule devise dans les colisages, elle est automatiquement sélectionnée pour vous faire gagner du temps. Si plusieurs devises sont présentes, vous devez choisir.

\textbf{Q11 : Que signifie "devise majoritaire" ?}

R : C'est la devise qui représente le plus grand pourcentage de la valeur totale du dossier. Un avertissement s'affiche si vous choisissez une autre devise.

\textbf{Q12 : Puis-je régénérer une note plusieurs fois ?}

R : Oui, vous pouvez régénérer autant de fois que nécessaire. La note existante est automatiquement supprimée avant la régénération.

\textbf{Q13 : Dans quel ordre sont triées les lignes de la note ?}

R : Les lignes sont automatiquement regroupées et triées par régime douanier selon cet ordre de priorité : 100\% DC, 100\% TR, EXO, Ratios DC, Ratios TR, TTC.

\textbf{Q14 : Pourquoi la colonne Qté Colis affiche 2 décimales ?}

R : C'est le format standard pour cette colonne, cohérent entre l'interface et les exports PDF/Excel/CSV.

\section{Questions sur les Exports}

\textbf{Q15 : Quelle est la différence entre les formats d'export ?}

R : 
\begin{itemize}[leftmargin=*]
    \item \textbf{PDF} : Document professionnel pour impression et archivage officiel
    \item \textbf{Excel} : Fichier tableur pour analyse et calculs complémentaires
    \item \textbf{CSV} : Format texte pour import dans d'autres systèmes
\end{itemize}

\textbf{Q16 : Puis-je exporter en anglais ?}

R : Oui, le format PDF est disponible en français et en anglais. Sélectionnez la langue lors de l'export.

\textbf{Q17 : Les exports contiennent-ils toutes les données ?}

R : Oui, tous les formats d'export contiennent les mêmes données : informations de pesée, synthèse par régime, taux de change et détails des lignes.

\chapter{Glossaire}

\section{Termes Métier}

\begin{description}[leftmargin=3cm, style=nextline]
    \item[Colisage] Ensemble des informations relatives aux colis d'une marchandise : poids, volume, quantité, valeur, etc.
    
    \item[Code HS] Code du Système Harmonisé (Harmonized System) : nomenclature internationale de classification des marchandises sur 8 chiffres.
    
    \item[Commissionnaire en douane] Professionnel agréé qui effectue les formalités douanières pour le compte de ses clients.
    
    \item[Conversion] Opération de change de devises à une date donnée avec des taux officiels.
    
    \item[DC (Droits de Douane)] Taxes perçues par les douanes sur les marchandises importées.
    
    \item[Dédouanement] Ensemble des formalités douanières nécessaires pour l'importation ou l'exportation de marchandises.
    
    \item[Devise] Unité monétaire d'un pays (USD, EUR, XOF, etc.).
    
    \item[Dossier] Ensemble documentaire regroupant toutes les informations relatives à une opération de transit.
    
    \item[EXO (Exonération)] Régime douanier permettant l'exemption totale de droits et taxes.
    
    \item[Note de détail] Document de synthèse récapitulant l'ensemble des marchandises d'un dossier avec leurs valeurs converties.
    
    \item[OT (Ordre de Transit)] Numéro d'identification d'une opération de transit.
    
    \item[Pré-dédouanement] Préparation des documents et calculs avant la déclaration en douane officielle.
    
    \item[Régime douanier] Statut juridique applicable à une marchandise (DC, TR, EXO, TTC).
    
    \item[Taux de change] Valeur d'une devise par rapport à une autre à une date donnée.
    
    \item[TR (Transit)] Régime douanier permettant le passage de marchandises sans paiement immédiat des droits.
    
    \item[TTC (Toutes Taxes Comprises)] Régime incluant tous les droits et taxes dans le calcul.
\end{description}

\section{Termes Techniques}

\begin{description}[leftmargin=3cm, style=nextline]
    \item[Application web] Logiciel accessible via un navigateur internet sans installation locale.
    
    \item[Badge] Élément visuel coloré indiquant un statut ou une catégorie.
    
    \item[Base de données] Système de stockage structuré des données de l'application.
    
    \item[CSV] Comma-Separated Values : format de fichier texte pour l'échange de données tabulaires.
    
    \item[Dialogue modal] Fenêtre qui s'affiche au-dessus du contenu principal et nécessite une action de l'utilisateur.
    
    \item[Export] Action de générer un fichier à partir des données de l'application.
    
    \item[Filtre] Outil permettant de restreindre l'affichage des données selon des critères.
    
    \item[Formulaire] Interface de saisie de données avec des champs à remplir.
    
    \item[HTTPS] Protocole de communication sécurisé pour le web.
    
    \item[Import] Action d'intégrer des données externes dans l'application.
    
    \item[Interface] Ensemble des éléments visuels permettant l'interaction avec l'application.
    
    \item[Next.js] Framework de développement web utilisé pour SFX PreDouane.
    
    \item[Notification] Message temporaire informant l'utilisateur du résultat d'une action.
    
    \item[Pagination] Division des résultats en plusieurs pages pour faciliter la navigation.
    
    \item[PDF] Portable Document Format : format de fichier pour documents électroniques.
    
    \item[Prisma] ORM (Object-Relational Mapping) utilisé pour la gestion de la base de données.
    
    \item[React] Bibliothèque JavaScript pour la construction d'interfaces utilisateur.
    
    \item[Responsive] Interface qui s'adapte automatiquement à la taille de l'écran.
    
    \item[Session] Période d'utilisation de l'application entre la connexion et la déconnexion.
    
    \item[SQL Server] Système de gestion de base de données Microsoft utilisé par l'application.
    
    \item[Tableau] Composant d'interface affichant des données sous forme de lignes et colonnes.
    
    \item[Toast] Notification temporaire qui apparaît brièvement à l'écran.
    
    \item[URL] Uniform Resource Locator : adresse web de l'application.
\end{description}
